## Debugging Approach

Use hypothesis-driven debugging:
1. **Observe** -- gather symptoms, error messages, and reproduction steps
2. **Hypothesize** -- form 2-3 likely causes ranked by probability
3. **Test** -- systematically verify or eliminate each hypothesis
4. **Conclude** -- identify root cause with evidence
5. **Recommend** -- provide specific fix with file paths and code changes

## Common Investigation Areas

### React / Frontend Issues

**Component rendering problems:**
```bash
# Check component imports and exports
grep -r "export.*default\|export.*const" --include="*.tsx" apps/admin-ui/src/resources/

# Check for missing providers
grep -r "Provider\|Context" --include="*.tsx" apps/admin-ui/src/app/

# Check for React key warnings
grep -r "key=" --include="*.tsx" apps/admin-ui/src/

# Check useEffect dependencies
grep -r "useEffect" --include="*.tsx" --include="*.ts" apps/admin-ui/src/
```

**Refine-specific issues:**
```bash
# Check resource registration
grep -r "resources\|ResourceProps" --include="*.tsx" apps/admin-ui/src/app/

# Check data provider configuration
grep -r "dataProvider\|graphqlDataProvider" --include="*.tsx" --include="*.ts" apps/admin-ui/src/

# Check hook usage
grep -r "useList\|useOne\|useForm\|useTable\|useNavigation" --include="*.tsx" apps/admin-ui/src/
```

**GraphQL / Data fetching issues:**

### Hasura / Database Issues

**Permission problems (data not showing):**
```bash
# Check Hasura metadata for permissions
find apps/nhost/nhost/metadata -name "*.yaml" | xargs grep -l "permissions\|select_permissions"

# Check specific table permissions
cat apps/nhost/nhost/metadata/databases/default/tables/itap_*.yaml

# Check role configuration
grep -r "x-hasura\|hasura-role" --include="*.yaml" --include="*.ts" apps/
```

**Migration issues:**
```bash
# List all migrations in order
ls -la apps/nhost/nhost/migrations/default/

# Check specific migration
cat apps/nhost/nhost/migrations/default/*/up.sql

# Check for conflicting migrations
grep -r "ALTER TABLE\|DROP" apps/nhost/nhost/migrations/default/*/up.sql
```

**Relationship issues:**
```bash
# Check object/array relationships in metadata
grep -r "object_relationships\|array_relationships" --include="*.yaml" apps/nhost/nhost/metadata/

# Check foreign key constraints in migrations
grep -r "FOREIGN KEY\|REFERENCES" apps/nhost/nhost/migrations/default/*/up.sql
```

## Common Root Causes (ITAP-specific)

1. **Hasura permissions silently filter data** -- query returns empty results, no error
2. **GraphQL fragment mismatch** -- fragment references fields not in Hasura schema
3. **graphql-codegen stale types** -- need to run `pnpm gql` after schema changes
4. **Refine resource name mismatch** -- `name` in resource config doesn't match Hasura table
5. **Nhost Auth token expired** -- silent failures in GraphQL requests
6. **Human library WASM not loaded** -- face recognition fails silently
7. **TailwindCSS v4 class differences** -- v3 classes not working in v4
8. **React 19 strict mode** -- double render causing unexpected behavior
